# Прогнозирование временных рядов

## Вводная часть

И так, проанализировав полученные данные, я увидел, что некоторые столбцы практически пусты, а некоторые заполнены на <30% (отобрал я столбцы у которых была заполненность >30%).

Также была установлена закономерность, что некоторые колонки зависят от других (особенно в моменты выбросов (см. рис.1)), либо повторяют друг друга (но не полностью (см. рис.2)).

У меня появилось две возможные задачи относительно этих данных (восстановить пропущенные столбцы (временные ряды), либо отобрать наиболее лучшие столбцы и уже работать с ними (предсказывать значения для них)).

Выбрал я вторую задачу, т.к было трудно сказать какая корреляция между столбцами и какова их природа.

За базовый горизонт я взял 1 минуту, т.к это оптимальный вариант, а также минимальный шаг моих данных, но другие варианты горизонтов представлены в artifacts/full_horizon (и сравнение горизонтов лежит в artifacts/figures).

Модели с базовым горизонтом сразу лежат в artifacts (col_* и figures и парочку csv).

**Python:** 3.12.

### Как устроен прогон (коротко)

1. Сортировка по времени, общая минутная сетка.
2. Несколько точек в одну минуту ->**среднее**.
3. Дыры на сетке ->линейная интерполяция, лимит `**INTERP_LIMIT_MINUTES`**.
4. На куске **train** по времени считаются нижняя/верхняя граница по квантилям, потом **clip** на весь ряд.
5. Деление **70% / 15% / 15%**: train / val / test по строкам признаков.
6. **Бленд**: перебор пар `(ridge,hgb)`, `(ridge,rf)`, `(hgb,rf)` и веса на val.
7. **Walk-forward** на train+val: несколько окон, средний val MAE; если данных мало — берём один val.
8. **Победитель** среди ridge / hgb / rf / blend по правилам выше; при равенстве порядок: ridge ->hgb ->rf ->blend.

В `**metrics.json`** есть, например: `sv_winner`, `wf_used`, `wf_n_folds`, ошибки на test у выбранной стратегии.

### Интерпретация, выводы и рекомендации

**С чего смотреть.** После обучения открыть `**artifacts/summary_metrics.csv`** и при необходимости `**artifacts/winners_by_column.csv**`: по каждой колонке видно, кто выиграл (`winner`, `winner_label`), и сырые MAE по train / val / test для ridge, hgb, rf и blend.

**Два способа выбрать победителя.** В `metrics.json` поле `**sv_winner`** — лучший вариант по **одному** отрезку val. Поле `**winner`** — итог после правила с **walk-forward** (средний val MAE по нескольким окнам), если `**wf_used`** true; иначе walk-forward не применился, и выбор совпадает с одним val. Флаг `**wf_disagrees_sv**` значит: «лучший по одному val» и «лучший по walk-forward» — разные модели. Тогда смотреть вместе `**test_mae_winner**` (ошибка на test у итогового `winner`) и `**test_mae_at_sv_winner**` (какая была бы ошибка на test, если бы зафиксировать `sv_winner`). Если вторая заметно лучше — итоговый выбор стоит обсуждать отдельно (val мог «повезти»).

**Про `horizon_sweep` и `horizon_full_sweep`.** Это вспомогательные прогоны «как меняется ошибка, если цель сдвинута на h минут». Они не заменяют основную сводку по `pipeline` для постановки с горизонтом 1.

## Прогноз (без переобучения)

Нужны папки `artifacts/col_<n>/` после обучения.

```
python -m src.predict --data путь_к_файлу.csv
```

- `--artifacts` — где лежат `col_*` (по умолчанию `artifacts/`).
- `--out` — куда писать `pred_col_*.csv` (по умолчанию `artifacts/predict_run/`).
- `--cols 15 20` — только эти колонки (иначе все из `TARGET_COLS`).

В CSV колонка `**pred_pick**` — это выбранный на обучении вариант и бленд из `strategy_meta.joblib`. Границы обрезки те же, что в meta (`winsor_lo`, `winsor_hi`).

---

### Лёгкий sweep по горизонту (Ridge, опционально HGB)

Таргет сдвигается на `h` минут вперёд, на test считается MAE.

```
python -m src.horizon_sweep
python -m src.horizon_sweep --cols 15 16 --horizons 1 60 240 --with-hgb
```

Результат: `**artifacts/horizon_sweep.csv**`. Колонка `**test_mae_last_value**` — наивный прогноз «как последнее значение». Горизонты по умолчанию — в `src/config.py`.

### Полный прогон по горизонтам (как в `pipeline`)

Те же три модели, бленд, walk-forward, но для каждого горизонта и своей папки артефактов:

```
python -m src.horizon_full_sweep --run-tag имя_прогона --grid normal
```

- `--grid normal` (или `targets`) — только `**TARGET_COLS**` и полный набор горизонтов из `src/config.py`.
- `--grid data` — все числовые колонки файла (часто с пустыми; для отладки).

Пишет в `**artifacts/full_horizon/<run-tag>/**`.

## Настройка (`src/config.py`)

- `DATA_PATH`, `TARGET_COLS` (в CSV колонка **0** — время).
- Лаги, окна rolling, доли train/val, интерполяция, `CLIP_LOW_Q` / `CLIP_HIGH_Q`.
- `HGB_PARAMS`, `RF_PARAMS`, `RIDGE_ALPHA`, `BLEND_PAIRS`.
- Walk-forward: `WF_VAL_FRAC`, `WF_MIN_TRAIN_FRAC`, `WF_MAX_FOLDS`, `WF_MIN_FOLDS`.
